# Next on Wembley

A small web app that recommends TV shows to a two-person household. It tracks
what each partner has watched, is watching, dropped, or wants to watch, then
produces three ranked recommendation lists generated by Claude:

1. **Co-watch (Global)** — shows both partners would likely enjoy together
2. **Corey's Picks** — personalized for one partner
3. **Jaimie's Picks** — personalized for the other

Recommendations cover both **new shows** and **continuations** of in-progress
series. Each partner can vote Agree / Disagree / Maybe on every recommendation;
votes are visible to the other partner and influence future rankings. The LLM
reasons over watch history, ratings, in-progress shows, active streaming
subscriptions, regional streaming availability, TMDb community ratings, and
episode counts.

## Stack

- **Next.js 16** (App Router, React 19, Server Actions)
- **Prisma** ORM — SQLite for local dev, Postgres in production
- **iron-session** — passcode auth with HTTP-only cookie sessions
- **Anthropic API** (`@anthropic-ai/sdk`) — recommendation generation
- **TMDb API** — show metadata, posters, streaming providers
- **Tailwind CSS v4** + **Radix UI**
- **Vitest** + Testing Library

## Getting started

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment

Copy the template and fill in the values:

```bash
cp .env.example .env.local
```

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | Connection string. `file:./dev.db` for local SQLite. |
| `SESSION_SECRET` | iron-session cookie key — 32+ random chars (`openssl rand -base64 32`). |
| `ANTHROPIC_API_KEY` | Claude API key for recommendation generation. |
| `TMDB_API_KEY` | TMDb v3 API key — register at themoviedb.org/settings/api. |
| `SEED_COREY_PASSCODE` / `SEED_JAIMIE_PASSCODE` | Login passcodes set by the seed script. |

Prisma's CLI also reads `DATABASE_URL` from `.env` — keep `file:./dev.db` there
for local dev.

### 3. Set up the database

```bash
npm run db:migrate   # apply migrations
npm run db:seed      # create the two household users
```

### 4. Run the dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and log in with a passcode.

## Deployment

Deployed on Vercel with Postgres. The `vercel-build` script runs
`prisma migrate deploy` and the seed script on each build, so
`SEED_COREY_PASSCODE` and `SEED_JAIMIE_PASSCODE` must be set as project
environment variables.
